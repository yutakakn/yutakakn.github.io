  • ルール
      　実行画面に示したように表現された連立方程式を解く「加算クロスワード」 　です。つまり、 　
        　　
      1. 三角の中の数字はその列（または行）の総和。 　　
      2. 各空白には1から9までの数字が重複せず入ります!
        　　　つまり、空白のマスの個数は最大9個で、総和の最大は45です。 　

      　を満たします。 　右のボタンから1から9の数字を選びマウスでクリックするとコマを置くことが 　できます。すべてのコマを埋め、かつ正解であれば次のステージへ進めます!

  • 遊び方
      操作はすべてマウスで行ないます。
      それぞれの数字は右のラジオボタンから選択し、マウスの左クリックで配置 します。置いた数字を消したい場合は、マウスの右クリックをします。 Macintoshなどではマウスボタンが１つしかありませんので、METAキーを押し ながらクリックすると、同じ効果が得られます。
      最初の盤面に戻したい場合は、クリアボタンを押します。

      キーボードにも対応しました。数字の「１」～「９」を押すと番号を セレクトできます。

    • 流れ
      1. Windows95のDOS窓で Turbo C++ 4.0J を使って、パズルを解くプログラムを 作ります。この作業でデータ構造を固めます。
      2. 次にJavaでプログラミングするのですが、その前に作るアプレットの 状態遷移図みたいなのを紙に書いて考え、仕様を練ります。
      3. 紙に書いたメモを見ながら実際にJavaを書きます。あまり、ディスプレイ の前が考え込む...ということは少ないですね。
      4. 動くだけのものができてくると、後はいろいろな機能を付けたりして自分 のイメージにあったものを作っていきます。ここからがプログラミングで一番 楽しいところですね。
      5. できたプログラムをFDに入れて学校に持ってきます。ftpでサーバへ転送 してWWW上で公開します。念のため、Internet ExplorerとNetscapeの両方で チェックします。(Operaはどうしようかな:-))
      6. CGIランキングを用意して、とあるHPに登録してもらって完成！

  • あとがき(1998.1.7)
      以前から作ろうと思っていたパズルなんですが、ナンプレやお絵かきロジックに 比べてデータ構造がややこしいので見送っていました。この冬休み（といっても それほど暇があったわけではないのですが）に構想を練っていました!しかし、 やはりデータ構造の複雑さに参っていたところ、図書館で見つけた「ナノピコ教室」 という本にカックロを解くプログラムが載っていたのです。これで道が見えてきた ...と思っていたら、なんと言語がＢＡＳＩＣ。もちろん、Visual Basicではなく、 昔のＮ88-ＢＡＳＩＣ。とても、見やすいとは言えず（失礼）四苦八苦。 とりあえず、どういうデータ構造にすればいいかだけが分かったのみで、後は なんとか自己流で自動解答プログラムを作りました!思ったよりも1秒未満で解が でるので、作業は一気に進むことになります。
